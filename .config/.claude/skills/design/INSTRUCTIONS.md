# Design Guidelines Skill

## 概要

このスキルは、AI生成UIにありがちな「分布収束」による没個性化を防ぎ、
ブランドに沿った独自性のあるデザインを実現するためのガイドラインを提供する。

## 問題: なぜAI生成UIは似通うのか

LLMは学習データの中で「最も確率の高い選択肢」を返す性質がある。
これにより、以下のような統計的に安全な選択に収束してしまう:

- **Inter / Roboto** フォント
- **紫グラデーション** (`#667eea` → `#764ba2`)
- **汎用青ボタン** (`#007bff`, `#3b82f6`)
- **テンプレート的なカードグリッド**

これは「AI slop」と呼ばれ、一目で「AIが作った」と分かる見た目になる。

## 解決策: 意図的な分布からの逸脱

このスキルでは以下のアプローチを取る:

1. **禁止事項の明確化**: 統計的に選ばれやすい選択肢を明示的に除外
2. **具体的な代替案の提示**: 禁止事項の代わりに使うべきものを具体化
3. **チェックリスト**: 生成後の品質確認

## 使用タイミング

以下の作業時にこのスキルを読み込む:

- React / Vue / HTML でのUIコンポーネント生成
- Tailwind CSS でのスタイリング
- カラーパレットやテーマの設計
- アニメーション/インタラクションの実装

## ガイドラインファイル

| ファイル | 内容 |
|---------|------|
| `typography.md` | フォント選択ルール |
| `theme.md` | カラーとビジュアルスタイル |
| `motion.md` | アニメーション原則 |
| `background.md` | 背景デザインルール |
| `anti-patterns.md` | 避けるべきパターン集 |

## クイックチェックリスト

UI生成後に以下を確認:

- [ ] 禁止フォント（Inter, Roboto, Open Sans）を使っていない
- [ ] 紫グラデーションを使っていない
- [ ] 汎用青ボタン（#007bff）をそのまま使っていない
- [ ] レイアウトに独自性がある
- [ ] アニメーションが過度でない（500ms以下）
- [ ] ブランドの個性が表現されている

## 推奨カラーパレット（クール系）

```css
:root {
    --primary: #0D9488;      /* ティール */
    --secondary: #134E4A;    /* ダークティール */
    --accent: #10B981;       /* エメラルド */
    --background: #F0FDFA;   /* ミント */
    --foreground: #1E293B;   /* スレート */
}
```

## 技術スタック

このスキルは以下を想定:

- **フレームワーク**: React
- **スタイリング**: Tailwind CSS
- **アニメーション**: Framer Motion (オプション)

## 基本原則

### 1. 禁止事項を最初に確認

```
避けるべきフォント: Inter, Roboto, Open Sans, Arial
避けるべきカラー: 紫グラデーション、汎用青(#007bff)
避けるべきパターン: テンプレート的カードレイアウト
```

### 2. ブランドの文脈を理解

プロジェクト固有のブランドガイドラインがあれば、それを優先する。
なければ、このスキルのデフォルト推奨（ティール/エメラルド系）に従う。

### 3. 具体例を参照

抽象的な指示より具体例を見ることで、適切な判断ができる。
各ガイドラインファイルにはReact + Tailwindのコード例を含む。
