# Competitive Programming Skill

## 概要

paiza、AtCoderなどの競技プログラミング問題を解くためのスキルセット。
問題分析からコード生成、最適化まで一貫してサポートします。

## 対応プラットフォーム

- **paiza**: レベル S, A, B, C, D
- **AtCoder**: ABC, ARC, AGC
- **その他**: LeetCode, Codeforces, yukicoder

## 対応言語

- **Python** - PyPyで高速化可能
- **JavaScript** - Node.js環境（ES6+モダン構文を使用）

## JavaScript コーディング規約

JavaScriptコード生成時は以下のモダンな書き方を必ず使用:

- `var` は使用禁止 → `const` / `let` を使用
- `function()` より アロー関数 `() =>` を優先
- 分割代入 `const [a, b] = ...` を積極的に使用
- `Array.from()`, `new Array().fill()` で配列初期化
- `for`ループより `map`, `filter`, `reduce`, `every`, `some` を優先
- スプレッド構文 `...arr`, `[...set]` を活用
- テンプレートリテラル `` `${var}` `` を使用

## 使用タイミング

以下の作業時にこのスキルを参照:

- 競技プログラミングの問題を解くとき
- アルゴリズムの選定に迷ったとき
- TLE/MLEが発生したとき
- 典型アルゴリズムのテンプレートが必要なとき

## クイックスタート

### 1. 問題を解く

問題文を提示して、解答コードを生成:

```
この問題を解いてください:
[問題文を貼り付け]
使用言語: Python
```

### 2. 詳細分析

複数の解法を比較検討したい場合:

```
この問題の解法を分析してください:
[問題文を貼り付け]
```

### 3. 最適化

TLE/MLEが発生した場合:

```
このコードを最適化してください:
[コードを貼り付け]
問題: TLE
```

## 含まれるテンプレート

### 入出力パターン
- Python: `sys.stdin.readline`、高速化設定
- JavaScript: 同期入力パターン

### 典型アルゴリズム
- 二分探索（めぐる式）
- Union-Find
- BFS/DFS
- Dijkstra
- 累積和（1D/2D）
- 素数関連（エラトステネス、素因数分解）
- mod演算（繰り返し二乗法、逆元、組合せ）

## 計算量早見表

```
N ≤ 10   → O(N!)       可能
N ≤ 20   → O(2^N)      可能
N ≤ 100  → O(N^3)      可能
N ≤ 3000 → O(N^2)      可能
N ≤ 10^5 → O(N log N)  必要
N ≤ 10^6 → O(N)        必要
N ≤ 10^8 → O(log N)    必要
```

## 典型アルゴリズムチェックリスト

問題を見たときに確認:

- [ ] 全探索で間に合うか？
- [ ] 二分探索が使えるか？（単調性）
- [ ] 累積和/いもす法で区間処理できるか？
- [ ] DPで解けるか？（最適部分構造）
- [ ] グラフとして捉えられるか？
- [ ] 貪欲法が正当化できるか？
- [ ] 数学的な公式で解けるか？
