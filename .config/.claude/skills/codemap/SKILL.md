## 使用タイミング

- `/update-codemaps` コマンドで明示起動
- 新しいプロジェクトに初めて取り組む際のコードベース理解
- 大きなリファクタリングや機能追加の後

## 機能概要

1. **プロジェクト構造スキャン**: git リポジトリルートを基点に、プロジェクトタイプ・entry surfaces を自動判定
2. **architecture.md 生成**: トークン効率的（1000トークン以下）なアーキテクチャドキュメントを `docs/CODEMAPS/` に生成
3. **差分検知と更新**: 既存 codemap がある場合、セクション単位で差分を比較し、大きな変更はユーザー承認を求める
4. **陳腐化チェック**: git commit ベースで codemap の鮮度を判定し、再生成を推奨

## カスタマイズ

プロジェクトルートに `.codemap.yml` を配置することで、出力先・token budget・除外パターンを上書き可能。

詳細な手順は `INSTRUCTIONS.md` を参照。
