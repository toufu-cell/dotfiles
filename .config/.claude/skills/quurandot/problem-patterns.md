# Quurandot 問題パターン別攻略法

各問題タイプの特徴と、効果的なアプローチを解説する。

## パターン1: テキスト推理型

**特徴**: 文章で概念・法則・人物などを問う。一見普通のクイズに見えるが、検索で効率的に解くことが求められる。

**攻略法**:
1. 問題文のキーフレーズを抽出
2. そのまま検索 → AIによる概要で答えが出ることもある
3. ヒットしない場合は表現を言い換えて再検索
4. 英語での検索も並行して試す

**例: Q1-1（パーキンソンの凡俗法則）**
- 問題: 「取るに足らない議題であるほど議論が白熱する」ことのたとえに用いられる施設
- アプローチ: 「些細 議題 議論 法則」等で検索 → 凡俗法則 → 自転車置き場

**注意点**:
- AI概要で答えが出ても、元の情報源で **裏取り** すること
- 「ほぼ正解だが微妙に違う」解答に注意（例: 「駐車場」は誤答、「自転車置き場」が正解）

## パターン2: 画像特定型

**特徴**: 写真・彫刻・建造物などの画像から対象を特定する。加工（モノクロ化、切り抜き、輪郭抽出）が施されていることが多い。

**攻略法**:
1. 画像の特徴を言語化する（材質、形状、時代感、文化圏など）
2. Googleレンズ / 画像検索で直接検索
3. ヒットしない場合、推測したキーワードを追加
4. AIに「何に見えるか」を聞いて方向性を得る

**例: Q3-2（馬房 / 競馬場）**
- 問題: 加工された建物の写真
- アプローチ:
  - 画像検索 → 直接ヒットしない
  - AIに聞く → 「馬小屋に見える」
  - 「馬房 12A」で検索 → 具体的な場所を特定

**例: Q0-2（ファルネーゼのアトラス）**
- 問題: 石の彫刻の一部
- アプローチ: 画像検索 + "fish carving" → 古代ローマの彫像と判明

**テクニック**:
- **AIによる着色**: モノクロ画像をAIにカラー化してもらい、再度画像検索
- **部分拡大**: 特徴的な箇所をクロップして検索
- **逆引き**: 特徴から推測した対象の画像を検索し、元画像と照合

## パターン3: 形・図形解読型

**特徴**: 幾何学的な図形、回路図、平面図などの意味を問う。「何を表しているか」を当てる問題。

**攻略法**:
1. 図形の特徴を分析（対称性、構成要素の数、配置パターン）
2. 何のビュー（上面図、断面図、回路図など）かを推測
3. 推測に基づいた検索キーワードを構築
4. ドメイン知識を追加して絞り込む

**例: Q1-2（建築物の平面図）**
- 問題: 正三角形・正方形などの図形の組み合わせ
- アプローチ:
  - 建築物の上面図と推測
  - 「正三角形 正方形 建築」「美術館 正三角形 航空写真」
  - → 特定の建物に一致

**例: Q1-3（合成抵抗の回路図）**
- 問題: 複雑な回路図
- アプローチ:
  - 構成要素の数に着目 → 「抵抗 21個」で検索
  - 正方形分割正方形問題との対応を発見

**例: Q4-2（エッフェル塔 × 凱旋門）**
- 問題: 2つの図形に共通する6文字
- アプローチ:
  - 左図の星型 → 上からの図 → 方角情報からエッフェル塔と特定
  - 右図 → 凱旋門と特定
  - 両方に刻まれた人物名 → ラザール・カルノー

**キーワード構築のコツ**:
- 図形の数や配置: 「21個」「正方形分割」「5x5」
- 用途の推測: 「建築」「回路」「地図」「旗」
- 上から見た図かもしれないという発想を持つ

## パターン4: 暗号・符号解読型

**特徴**: 一見ランダムな文字列や未知の文字体系を解読する。入力方式の変換が鍵になることが多い。

→ 詳細は `decoding-techniques.md` を参照

**攻略法**:
1. 文字列の特徴を分析（使用文字種、長さ、パターン）
2. 部分的にでも意味のある単語が見つかるか試す
3. 入力方式の変換を疑う（注音、倉頡、他のIME）
4. 画像検索で似た字形を探す

**例: Q0-3（注音輸入法）**
- 問題: `w96j0yjo4el2k7g0g4s83uyji4`
- アプローチ: `w96j0` で検索 → 台灣がヒット → 注音輸入法と判明

**例: Q3-3（倉頡輸入法 × カラーコード）**
- 問題: 漢字 + 色が付いた表示
- アプローチ:
  - カラーピッカーでカラーコードを特定
  - 「骨 BBB」等で検索 → 倉頡輸入法と判明

## パターン5: パターン・対応型

**特徴**: 数字・日付・記号の並びからパターンを見つけ、欠落した要素を特定する。

**攻略法**:
1. 既知の要素を整理（何がわかっていて何がわからないか）
2. 要素間の関係性を探る
3. 部分的な対応関係を検索で確認
4. パターンの法則を特定して欠落要素を導出

**例: Q2-1（覆面算）**
- 問題: 26マスのグリッドに数字
- アプローチ:
  - 26 → アルファベットと対応
  - 数字を文字に変換 → 「OMY ENDRS」→ 有名な覆面算SEND MORE MONEY

**例: Q2-2（ユールラッズ）**
- 問題: 12/12〜12/24の各日にアルファベット1文字
- アプローチ:
  - 「クリスマス 13日間」で検索
  - アイスランドのユールラッズ（13人のサンタクロース）

**例: Q3-1（歌詞の視覚化）**
- 問題: ユニオンジャック、チェック、マーブル模様などの視覚要素の列
- アプローチ:
  - 模様を言語化して組み合わせ検索
  - 歌詞の視覚的表現と気づく → 楽曲を特定

## パターン6: 複合型

**特徴**: 複数の知識領域や解法テクニックを組み合わせる必要がある最も難易度が高いタイプ。

**攻略法**:
1. 問題を構成要素に分解する
2. 各要素を個別に調査
3. 要素間のつながりを発見
4. 全体像を統合して解答

**例: Q4-2（エッフェル塔 × 凱旋門 × ラザール・カルノー）**
- 要素1: 星型の図形 → エッフェル塔の上面図
- 要素2: 格子状の図形 → 凱旋門の壁面構造
- 要素3: 赤い点 → 刻まれた人物名の位置
- 統合: 両方に名前が刻まれた人物 → ラザール・カルノー

**例: Q4-3（バベルの図書館）**
- 要素1: `1/19560399...` → 巨大数（ボルヘス数）
- 要素2: `410` → Wikipediaの記事にバベルの図書館への言及
- 要素3: アルファベット + コンマ + ピリオド = 25文字
- 統合: バベルの図書館を再現したWebサイトの1冊目

## 難易度別アプローチのまとめ

### 易（Vol.0レベル）
- そのまま検索、画像検索、AI直接質問で解ける
- 1つの検索手法で完結することが多い

### 中（Vol.1-2レベル）
- 言い換え検索、完全一致検索の活用が必要
- 想像力を働かせて検索対象を絞る能力が求められる
- 複数の検索手法を組み合わせる

### 難（Vol.3-4レベル）
- 検索方法そのものを検索で見つける必要がある場合も
- 複数の知識領域を横断
- 部分的な手がかりから全体像を推論する力が求められる
- 粘り強い反復的な調査が不可欠
