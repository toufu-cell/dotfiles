# Quurandot 検索戦略ガイド

問題の種類に応じた最適な検索アプローチを体系化したガイド。

## 1. テキスト検索

### 1.1 キーワード検索（基本）

問題文や画像中のテキストからキーワードを抽出し、検索エンジンで検索する最も基本的な手法。

**手順**:
1. 問題文から固有名詞・数字・特徴的なフレーズを抽出
2. そのまま検索してヒットするか確認
3. ヒットしない場合は **言い換え** を繰り返す

**言い換えのコツ**:
- 抽象的な表現 → 具体的な表現に変換
- 例: 「取るに足らない議題で議論が白熱する」→「些細な問題 議論 長引く 法則」
- ドメイン知識を追加: 「建築」「美術館」「航空写真」など分野を絞る語を付与

**実例**（Q1-1）:
```
✗ "取るに足らない議題で議論が白熱する 施設"
○ "些細な問題 議論 白熱 法則"
○ "trivial topic heated discussion law"
→ パーキンソンの凡俗法則 → 自転車置き場
```

### 1.2 完全一致検索（"ダブルクォート"）

特定のフレーズをそのまま検索したい場合に有効。一般的なキーワードで埋もれてしまう場合に特に重要。

**有効な場面**:
- 一般的な単語の組み合わせが別の文脈でヒットしてしまう場合
- 特定の固有名詞やフレーズが含まれることが確実な場合

**実例**（Q4-1）:
```
✗ 大福 しお豆 ぱたもち  → 和菓子の検索結果ばかり
○ "大福" "しお豆" "ぱたもち" → クライミングルート名がヒット
○ "ぱたもち" 傾斜        → ロッククライミング関連がヒット
```

### 1.3 言語横断検索

問題が特定の言語や文化に関連する場合、その言語で検索する。

**実例**（Q0-1）:
```
"Speranza" "Sangue" → Turandot（トゥーランドット）
```

### 1.4 数値・コード検索

数字の列や特殊なコードが手がかりとなる場合。

**表記の工夫が重要**:
- 桁区切りを変える: `19560399` → `1.9560399` → `1,9,5,6,0,3,9,9`
- 専門データベースの活用: OEIS（オンライン整数列大辞典）など

**実例**（Q4-3）:
```
✗ "19560399"           → ヒットしない
○ "1.9560399"          → 巨大数研究wikiがヒット
○ OEIS: 1,9,5,6,0,3,9,9 → バベルの図書館関連
```

## 2. 画像検索

### 2.1 Google画像検索 / Googleレンズ

写真や図形から元の対象を特定する場合に使用。

**有効な場面**:
- 彫刻・建造物・場所の写真が提示されている場合
- 特殊な文字・記号が含まれる場合

**注意点**:
- 加工された画像（切り抜き・モノクロ化・輪郭抽出）は直接ヒットしにくい
- テキスト検索と組み合わせて補完する

**実例**（Q0-2）:
```
画像検索 + "mermaid" "fish carving"
→ ファルネーゼのアトラスの天球儀の一部 → いるか座
```

### 2.2 画像検索 + キーワード補助

画像だけではヒットしない場合、推測したキーワードを追加する。

**実例**（Q1-2）:
```
画像検索 + "正三角形 正方形 建築"
画像検索 + "美術館 正三角形"
画像検索 + "航空写真 特徴的な形の建物"
→ 建物の航空写真と一致
```

### 2.3 AI による画像解析の活用

画像をAI（Gemini、ChatGPTなど）に入力して、何が写っているかの手がかりを得る。

**実例**（Q3-2）:
```
AIに画像を入力 → 「馬小屋に見える」という回答
→ "馬房 12A" で検索 → 具体的な場所を特定
```

**発展的なテクニック**:
- AIに画像を着色してもらってから画像検索
- AIに「何に見えるか」を聞いて検索キーワードのヒントを得る

## 3. Wikipedia・専門データベース活用

### 3.1 Wikipedia内検索

特定の数字や日付がWikipediaの記事に関連している場合がある。

**実例**（Q4-3）:
```
Wikipedia「410」の項目 → バベルの図書館への言及を発見
```

### 3.2 日付・期間の検索

**実例**（Q2-2）:
```
"12月12日から24日" "クリスマス 13日間"
→ アイスランドのユールラッズ
```

### 3.3 専門Wiki・データベース

- **巨大数研究wiki**: 大きな数字の特定
- **OEIS**: 数列の特定
- **世界の特殊文字wiki**: 特殊文字・人工文字の特定
- **個人制作フォント集**: 非標準的な文字体系の調査

## 4. AI（生成AI）の活用

### 4.1 直接質問

問題文をそのままAIに入力して回答を得る。近年の技術進歩により有効性が向上。

**注意**: AIの回答は検証が必要。必ず別の情報源で裏取りすること。

### 4.2 手がかり獲得のための活用

直接答えを聞くのではなく、解法の方向性を探るために使う。

- 「この画像は何に見えますか？」
- 「この文字列はどの言語/暗号体系ですか？」
- 「この数列に関連する概念は何ですか？」

## 5. 検索戦略の選択フローチャート

```
問題を確認
├── テキストが主体
│   ├── 固有名詞・特定フレーズあり → 完全一致検索
│   ├── 概念・法則を問う → キーワード言い換え検索
│   └── 特殊な文字列 → 暗号/符号として調査
│
├── 画像が主体
│   ├── 写真（実物） → Googleレンズ / 画像検索
│   ├── 図形・シェイプ → イメージを言語化 + キーワード検索
│   └── 特殊文字・記号 → 画像検索 + 文字体系の調査
│
├── 数字・日付が主体
│   ├── 日付パターン → 文化・イベントとの対応を検索
│   ├── 数列 → OEIS / 数学データベース
│   └── 大きな数字 → 表記を工夫して検索
│
└── 上記で解けない
    ├── 検索方法そのものを検索する
    ├── AIに手がかりを聞く
    └── 部分的な情報から仮説を立てて検証
```
