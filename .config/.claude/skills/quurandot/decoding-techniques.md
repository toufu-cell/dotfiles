# Quurandot 暗号・符号・特殊入力方式の解読テクニック

一見ランダムに見える文字列や未知の文字体系を解読するための手法集。

## 1. 入力方式の変換

### 1.1 注音輸入法（Zhuyin / Bopomofo）

台湾で主に使用される中国語入力方式。QWERTYキーボードの各キーに注音符号が割り当てられている。

**特徴**:
- 英数字の文字列が実は中国語のローマ字キー入力
- 有名な例: `ji32k7au4a83` = 「我的密碼」（私のパスワード）

**解読手順**:
1. 文字列の一部を検索して台湾との関連を確認
2. 注音キーボード配列表を入手
3. 各文字をキー配列に従って注音符号に変換
4. 注音符号から漢字に変換

**実例**（Q0-3）:
```
w96j0 → 台灣
完全な文字列を注音入力で変換 → 玉山（台湾の最高峰）
```

### 1.2 倉頡輸入法（Cangjie）

香港・マカオで主に使用される中国語入力方式。漢字の構成要素（字根）をアルファベットキーに対応させている。

**特徴**:
- 漢字の部品（偏旁）がアルファベットに対応
- 例: 「日」= A、「月」= B、「金」= C
- 部品の組み合わせで漢字を入力

**解読手順**:
1. 漢字と対応するアルファベットの組み合わせに注目
2. 「倉頡輸入法」「倉頡 コード」で検索
3. 対応表を参照して変換

**実例**（Q3-3）:
```
漢字 + カラーコード（短縮形）
カラーピッカーでコード特定 → 「骨 BBB」で検索 → 倉頡輸入法
```

### 1.3 その他の入力方式

以下の入力方式も可能性として頭に入れておく:
- **ピンイン入力**: 中国大陸で主流（ローマ字ベース）
- **五筆字型**: 字形ベースの入力方式
- **嘸蝦米輸入法**: 台湾の形状ベース入力方式
- **各言語のIME**: 韓国語（2ボル式）、日本語（ローマ字入力）など

## 2. 暗号解読

### 2.1 単純置換暗号

**チェックポイント**:
- アルファベット ↔ 数字の対応（A=0, B=1, ... or A=1, B=2, ...）
- シーザー暗号（固定値のシフト）
- 特定のマッピングテーブル

**実例**（Q2-1 - 覆面算）:
```
26マスのグリッド → 26文字 = アルファベット
数字を対応するアルファベットに変換
→ "OMY ENDRS" → SEND MORE MONEY（覆面算）
```

### 2.2 人工文字・創作文字

既存の文字をベースにした人工文字体系。

**調査リソース**:
- 世界の特殊文字wiki
- 個人制作のフォント集
- 新国字運動関連の資料
- Omniglot（世界の文字体系データベース）

**解読手順**:
1. 字形の特徴を観察（画数、対称性、構成パターン）
2. 既存の文字体系との類似性を探る
3. 画像検索で似た字形を検索
4. 文の構造から言語を推測（助詞の位置、語順など）

**実例**（Q2-3）:
```
日本語ベースの人工文字
- 画像検索で似た字形を発見
- 文字の並びから「この」「なに」等の日本語パターンを推測
- 世界の特殊文字wikiで特定
```

### 2.3 ポリュビオスの暗号表

5x5のグリッドにアルファベットを配置し、行列番号で文字を表す古典的暗号。

**使用場面**:
- グリッド状の配置 + 数字の組み合わせが見られる場合
- 座標系で文字を指定するパターン

## 3. カラーコード

### 3.1 短縮カラーコード

3桁の16進カラーコード（`#RGB`）が手がかりになる場合がある。

**解読手順**:
1. カラーピッカーツールで正確な色を取得
2. 16進数に変換
3. コードと他の要素（文字、記号）の対応関係を検索

**注意**: 画像圧縮で色が変わることがある。元画像の入手が重要。

**実例**（Q3-3）:
```
漢字に付けられた色 → カラーコードを特定（例: #BBB）
漢字 + カラーコード → 倉頡輸入法のキーコード
```

## 4. 数値系の解読

### 4.1 巨大数・特殊な定数

**調査リソース**:
- 巨大数研究wiki
- OEIS（オンライン整数列大辞典）
- Wolfram MathWorld
- Wikipedia の数字の個別記事

**検索のコツ**:
- 桁区切りを変えて検索: `19560399` → `1.9560399`
- OEISでは数列として検索: `1,9,5,6,0,3,9,9`
- Wikipediaでは個々の数（例: `410`）の記事を確認

### 4.2 日付・暦のパターン

- 特定の文化圏の祝祭日・行事と対応していないか
- 曜日や周期的なパターンがないか
- 暦の種類（グレゴリオ暦、ユリウス暦、旧暦など）

## 5. 解読の一般的フロー

```
未知の文字列/記号
│
├── 英数字のみ
│   ├── 短い（3-6文字） → カラーコード / 略語 / コード
│   ├── 規則的な繰り返し → 暗号・入力方式変換
│   └── ランダムに見える → 部分検索で手がかりを得る
│
├── 未知の文字体系
│   ├── 既知の文字に似ている → 人工文字・改変文字
│   ├── 完全に未知 → 画像検索 + 文字体系データベース
│   └── 構成要素がある → 部品ベースの入力方式
│
├── 数字のみ
│   ├── 少数の数字 → 置換暗号（アルファベット対応）
│   ├── 大きな数 → 巨大数データベース
│   └── 数列 → OEIS
│
└── 混合（文字 + 数字 + 記号）
    ├── グリッド配置 → ポリュビオス暗号 / 座標系
    ├── 日付 + 文字 → 暦・文化イベント対応
    └── 文字 + 色 → カラーコード対応
```

## 6. 判断のヒント: どの入力方式か

| 手がかり | 可能性の高い方式 |
|----------|------------------|
| 台湾との関連がある | 注音輸入法 |
| 香港・マカオとの関連 | 倉頡輸入法 |
| 漢字の部品が見える | 倉頡 / 五筆 |
| QWERTYキー配列上で意味がある | 注音 / その他のキーボード入力 |
| 文字列が中国語の音節に対応 | ピンイン入力 |
| パスワードのような英数字列 | 注音輸入法（ji32k7au4a83型） |
