# SKILL.md テンプレート: 自動発動型

サイズ目安: ~880B（上限 1200B）

```markdown
---
name: {skill-name}
description: "{スキルの目的を1文で}。{トリガー文脈を具体的に列挙}。{境界ケースでも使うべき場面を補足}"
---

## 発動条件

以下の表現が含まれたら必ずこのスキルを適用する:
- {発動条件1}
- {発動条件2}
- {発動条件3}

## 誤発動抑止（発動しない条件）

- {抑止条件1}
- {抑止条件2}
- {抑止条件3}

詳細な手順は `INSTRUCTIONS.md` を参照（このスキルのディレクトリ内）。
```

## Description の強度: 積極的（pushy、3-4文）

自動発動型は under-trigger しやすいため、description を積極的（pushy）に書く。
具体的なトリガー文脈を列挙し、境界ケースでも使うべき場面を補足する。

**良い例**: `"調査リクエストを受けて情報収集し、Obsidian にノートとして保存する。「調べて」「まとめて」「〇〇とは」という表現が含まれたら使用する。URLが指定された場合も使え。"`
**悪い例**: `"調査リクエストを処理するスキル"`（トリガー文脈が不明）

## 置換ルール

| プレースホルダ | 説明 |
|-------------|------|
| `{skill-name}` | スキル名（kebab-case） |
| `{スキルの目的を1文で}` | 50文字以内で簡潔に |
| `{トリガー文脈を具体的に列挙}` | 「〇〇したら」「△△という表現が含まれたら」等 |
| `{境界ケースでも使うべき場面を補足}` | 「〇〇の場合も使え」等 |
| `{発動条件N}` | ユーザー発話パターン（具体的な文言） |
| `{抑止条件N}` | 発動すべきでない文脈（具体的に） |

## 注意

- 発動条件は具体的な表現（「〇〇について調べて」等）で記述
- 誤発動抑止は必ず含める（コーディング文脈、明示的な拒否等）
- サイズ上限 1200B を厳守
