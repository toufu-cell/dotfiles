# SKILL.md テンプレート: コマンド起動型

サイズ目安: ~300B

```markdown
---
name: {skill-name}
description: "{スキルの目的を1文で}。{いつ使うかを1文で}"
---

詳細な手順は `INSTRUCTIONS.md` を参照（このスキルのディレクトリ内）。
```

## Description の強度: 簡潔（1-2文）

コマンド起動型は明示的に `/xxx` で起動するため、description は簡潔でよい。
ただし「目的」と「いつ使うか」の2点は必ず含める。

**良い例**: `"Docker コンテナの管理を支援する。/docker で起動"`
**悪い例**: `"Docker コンテナの管理を支援する"`（いつ使うかが不明）

## 置換ルール

| プレースホルダ | 説明 |
|-------------|------|
| `{skill-name}` | スキル名（kebab-case） |
| `{スキルの目的を1文で}` | 50文字以内で簡潔に |
| `{いつ使うかを1文で}` | コマンド名や使用場面を1文で |

## 注意

- コマンドファイル（`~/.config/.claude/commands/{skill-name}.md`）が必須
- SKILL.md 自体は最小限。ロジックは INSTRUCTIONS.md に書く
