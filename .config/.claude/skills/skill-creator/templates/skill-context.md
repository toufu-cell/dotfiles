# SKILL.md テンプレート: 文脈判断型

サイズ目安: ~300-625B

```markdown
---
name: {skill-name}
description: "{スキルの目的を1文で}。{具体的な作業名を列挙（例: React でのUI生成、フォーム実装、CSS レイアウト等）}"
---

## 使用タイミング

以下の作業時にこのスキルの INSTRUCTIONS.md を読み込む:
- {タイミング1}
- {タイミング2}
- {タイミング3}

詳細な手順は `INSTRUCTIONS.md` を参照（このスキルのディレクトリ内）。
```

## Description の強度: 中程度（2-3文）

文脈判断型は具体的な作業名を列挙して、適用場面を明確にする。

**良い例**: `"AI生成UIの没個性化を防ぐデザインガイドライン。React でのUI生成、フォーム実装、CSS レイアウト作業時に使用する"`
**悪い例**: `"デザインガイドライン"`（どの作業で使うか不明）

## 置換ルール

| プレースホルダ | 説明 |
|-------------|------|
| `{skill-name}` | スキル名（kebab-case） |
| `{スキルの目的を1文で}` | 50文字以内で簡潔に |
| `{具体的な作業名を列挙}` | 適用すべき具体的な作業名（3つ以上推奨） |
| `{タイミングN}` | 適用すべき作業コンテキスト |

## 注意

- 使用タイミングは具体的な作業名（「React でのUI生成」等）で記述
- 抽象的な条件は避ける
